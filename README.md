### **О проекте**

**Проект представляет собой экспериментальную платформу для торговли NFT с собственной системой аккаунтов и внутренними сделками. Пользователи могут создавать собственные NFT, обмениваться ими внутри площадки и взаимодействовать с интерактивными элементами интерфейса. Для администраторов реализован отдельный функционал управления платформой.**

**Проект создан скорее как арт обьект, демонстрирующий мой подход в разработке надёжных backend‑систем и реализации каких то стилевых решений на фронте. Проект использует базу данных PostgreSQL, где хранятся все данные о пользователях и NFT. Для работы с базой используется ORM Sequelize. Авторизация реализована через JWT, а логика API структурирована, оптимизирована и выдержана в едином архитектурном стиле.** 

**Спустя несколько месяцев после “релиза” проект пережил несколько рефакторингов — в первую очередь связанных с улучшением структуры базы данных и упрощением дальнейшего масштабирования. Изначально он был создан как итоговый проект в IT‑академии, но со временем превратился в самостоятельный учебный продукт, где торговля реализована абстрагированно от реальных финансовых операций.**

## **Технологии:**

- **Node.js + Express - основа серверной логики, с чётким разделением слоёв (роуты, контроллеры, сервисы).**
- **PostreSQL + Sequelize - Надежная база данных с собственным сервером, ORM обеспечивает безопасные запросы, миграции и удобную работу с моделями.**
- **JWT - механизм авторизации и контроля доступа к защищённым эндпоинтам.**

### **Структура проекта:**

`./app.js` - Главный файл проекта, она стартует сервер и аккумулирует в себе все используемые роутеры, имеет метод для отправки клиента и подключает cors.

`./routers/` - директория с js файлами, по которым распределены обработчики эндпоинтов.

`./controllers/` - директория с js файлами, которые несут в себе основные методы с логикой основных функций приложения и обработки запросов.

`./data/` - директория с js файлами, которые несут в себе логику для взаимодействия с базой данных PostgreSQL с помощью ORM Sequelize.

`./utils/` - директория с js файлами, которые содержат относительно простые методы, для различных задач, которые нужно решать сразу в нескольких местах проекта.

`./client/` - директория, содержащая в себе react клиент.

### API-эндпоинты:

**`/api/user/*`** 

- `POST “/signup”` - запрос для создания/входа в свой профиль, клиент получает JWT-токен.
- `GET “/getuserdata”` - запрос для получения данных о собственном профиле, защищен middleware для проверки JWT-токена.
- `GET “/addcoins”` - запрос для зачисления 10 единиц валюты пользователю, защищен middleware для проверки JWT-токена.

**`/api/platform/*`**

- `POST “/getcatalog”` - запрос для получения каталога NFT, к запросу можно приложить фильтры и получить отсортированный каталог.
- `GET “/getitem”` - запрос для получения данных о конкретной NFT.
- `GET “/getinv”` - запрос для получения собственного инвентаря NFT, защищен middleware для проверки JWT-токена.
- `PUT “/buynft”` - запрос для покупки NFT, присвоения ее пользователю-покупателю.
- `PUT “/putonsale”` - запрос для выставления на продажу одной NFT из инвентаря.
- `PUT “/putoutsale”` - запрос для отзыва с продажи своей NFT из инвентаря.
- `POST “/createnft”` - запрос для создания новой NFT.
- `GET “/getrand”` -  запрос для получения случайной NFT из базы.

**`/api/admin/*`**

- `DELETE “/killuser”` - запрос для удаления пользователя из базы, защищен middleware для проверки администратора.
- `GET “/confiscateuser”` - запрос для конфискации всех предметов у пользователя, защищен middleware для проверки администратора.
- `DELETE “/killnft”` -запрос для удаления NFT из базы, защищен middleware для проверки администратора.
- `GET “/confiscatenft”` -запрос для конфискации определенной NFT у пользователя, защищен middleware для проверки администратора.

### Безопасность:

Для обеспечения стабильной и безопасной работы сервера реализован ряд защитных механизмов, направленных на фильтрацию некорректных запросов, предотвращение перегрузок и защиту пользовательских данных.

- **CORS‑настройки** — ограничивают доступ к API только с доверенных источников, предотвращая несанкционированные запросы со сторонних доменов.
- **Middleware для валидации JWT и проверки администраторов** — проверяет корректность входящих запросов, защищая сервер от некорректных запросов.

### История проекта:

За время существования проект неоднократно переживал рефакторинги. Последнее крупное обновление — полная миграция с примитивного хранилища на SQLite3 и JSON‑файлы к полноценной PostgreSQL‑базе. Ранее NFT и связанные данные хранились в JSON, что подходило для раннего прототипа, но ограничивало масштабируемость и надёжность.

Теперь вся информация о пользователях, NFT и активности платформы находится в единой, мощной PostgreSQL‑базе, размещённой на отдельном сервере. Это позволило выстроить более строгую структуру данных, улучшить производительность и подготовить проект к дальнейшему развитию.
